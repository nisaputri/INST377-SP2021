Welcome to INST377!

This article is to help you get set up with a development environment that will let you test your own labs to see whether you're doing it right, before you submit your work. It's a separate article from the main lab assignments because some of you will have gotten into the course late, and will need a reference for setting up.

INST377 is a technical course about human problems. This means that, more than anything else, INST377 is about learning to be comfortable with new, potentially frustrating technical experiences. In this course, you will be given professional tools, which should help a lot with basic problems in learning to code. Those tools have some expectations with them, though, notably that you're willing to read a lot, install things, and be very comfortable with things simply not working the first time.

Most of programming is debugging and very carefully following instructions.

The following steps will be a little tricky, and need to be done in order, or you will not be able to submit your labs for this class. If you do not submit your labs, you can't get credit for them - but more importantly, you won't be able to practice your debugging before your group project.
